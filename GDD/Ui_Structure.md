# UI结构一览

## UI树状图
1. 主界面
   - 开始游戏
   - 继续游戏
   - 继续游戏+
   - 关卡图鉴（新增）
   - 输入密码
   - 沙盒关卡（临时）
   - 退出游戏
2. 战败界面
   - 重试
   - 返回主界面
3. 暂停界面
   - 继续游戏
   - 重试
   - 返回主界面
4. 通关界面
   - 文字”恭喜通关“
   - 返回主界面
5. 关卡图鉴（新增功能）
   - 10x8的类似相册一样的布局
   - 每个单元格内包含如下信息
     - 关卡名称
     - 关卡略缩图（只显示墙、地板、箱子、目标点、不显示玩家，如果玩家没有通过该关卡，则略缩图显示成问号）
     - 通关步数（玩家没通关则显示”未知“
     - 关卡密码（没有通过的关卡，显示”未知“）

### 关卡图鉴说明
1. 关卡图鉴玩家至少通过了第一关后才会显示在主界面上
2. 当玩家输入密码并通过了关卡，则图鉴内的对应的关卡也解锁，显示其通关步数和关卡密码
3. 一屏显示不下所有关卡，所以这一屏要做成可以上下滚动的
4. 左上角显示一个返回按钮，字体要24号，否则看不清。
5. 玩家多次游玩同一个关卡，需要在图鉴内的通关步数里记录他的最小步数，而不是最后一次通过的步数。

### 输入密码说明
1. 玩家输入了密码则瞬间进入关卡
2. 当玩家通过了输入密码的关卡则胜利后直接跳回主界面，不能继续游戏
3. 输入密码后，通过了关卡，玩家的“继续游戏”存档还是当时他自己玩的时候所在的关卡，输入密码只是用来解锁图鉴和体验没玩过的关卡的，不影响正常存档。他后续自己玩到曾经输入过密码的关卡则依然还需要再次挑战。

### UI通用规则
1. 关卡内所有文字必须大于24号字
