# 关卡生成思路
## 关卡生成的顺序
- 先生成剧情，剧情除了具体内容，还需要额外附加的2个维度的参数
  1. 压抑感：从0到1，越接近1，生成的地图路径越复杂，越接近0，则地图越空旷。
  2. 复杂度：从0到1，复杂度越高，能成功过关的方法就越少，失败的路径越多。
- 根据剧情的压抑感和复杂度进行评估并生成关卡

## 关卡的生成逻辑
### 概念
1. 推进步数比：过关需要推箱子的步数与关卡步数限制的比值。越高则关卡难度越高容错率越低。
2. 状态空间分支因子：在每一个合法状态下，玩家平均有多少种选择。分支越少，代表关卡越难。
3. 依赖深度：很多时候，要想推动b箱子，首先要先移动a箱子，推完b还要再把a箱子移动回原来位置。这种必须“暂时移开再移回”的箱子的数量影响了依赖深度。箱子越多，依赖深度越高。
4. 空间阻塞：箱子a挡住了去箱子b的路，你想要推b，必须先把a推开，但如果a被推到死角，关卡就废了。
5. 强制回溯：当推箱子尝试解密的时候，推了一步发现死锁，必须撤销这一步，回到上一个状态重新选。

### 整体思路
1. 首先，需要AI生成一个A Solver*，并输出“最短解决路径”、“状态搜索节点”、“强制回溯次数”，并将这三个参数分成三个难度等级。
10以内的最短解决路径就是简单、20-40就是中等、大于60步就是困难
100以内的状态搜索节点数就是简单、1000到5000是中等，大于50000是困难
0强制回溯数为简单、2-5为中等、10次以上为困难。

2. 生成关卡不要正向生成：
   - 先将箱子摆放在目标点
   - 尝试将箱子拉到各个点
   - 每拉动一次计算当前状态与目标状态的曼哈顿距离（最终需要衡量胃了让距离增加，我被迫做了多少次绕路的动作）
   - 如果拉动过程中导致了空间阻塞，则产生了先后顺序。
   - 如果遇到死锁，比如箱子被推到了推不动或者永远无法推到目标点的位置，则该生成路径作废。
  
3. 将所有能走通的路径进行评分，根据当前剧情需要的压抑感和复杂感选出符合的关卡即可。
4. 记录尝试过程中作废了多少条路径，也可以以此作为难度的评判。每次生成1个关卡，可以尝试100种推法，如果尝试1次就成功过关，那么关卡难度就是最低，如果尝试了99次才过关，意味着关卡里包含了大量死锁，那么就认为难度最困难。

### 从ai角度看的生成关卡的工作流
1. 生成剧情，并给出压抑和复杂维度的数值（根据剧情来打分）
2. 解析压抑和复杂度，量化成具体的参数，比如压抑0.8代表关卡里的墙必须占30%的单元格。
3. 根据逆向拉箱子的方法生成100个候选关卡
4. 用A*Solver跑这100个候选关卡，跳出难度最符合的关卡。（难度与回溯次数和复杂度相关。

为了尝试找到如何评价一个关卡难度到底是否合适，请暂时将关卡的压抑和复杂度以及你认为的难度等级写在关卡的左下角，我实际试玩后对你的打分进行修正即可。
